# 今晚吃什么 · 后端技术选型

承 [[1.0方案]] 的「技术形态（网页优先）」，把 1.0 后端**怎么做**定成一份可执行决策。同时收口 [[待澄清问题]] 的 Q1（模型选型）、Q3（存储）、Q4（身份）。

本文是**决策**，不是选项清单：每一项给唯一默认 + 一条「换路径」。模型价格为 2026-06 官网核实值，接入时以官网为准。

## 1 · 部署地域：大陆（总开关）

面向大陆用户，1.0 部署在国内云。这一项决定了后面所有选择，因为它同时定死两件事：

- **ICP 备案**：绑自有域名对大陆公开服务需备案（约 1–3 周）。**开发/内测阶段不阻塞**（可用云函数默认域名或本地联调），它只 gate「绑自有域名公开上线」，所以**备案与开发并行起步**即可。
- **模型 API 可达性**：从国内云出网，`api.anthropic.com`（Claude）不可靠可达。因此默认与兜底**都选国产模型**，Claude 不进 1.0 候选（见 §4）。

## 2 · 后端角色：无状态薄网关

[[待澄清问题]] 已定 Q4「先无登录」+ Q3「localStorage」⇒ **1.0 后端不持有任何用户态**。画像、历史都在浏览器。后端只是一个**薄网关**，单职责：

```
收前端 JSON { 画像, 今日早/午餐, 当下需求 }
  → 拼 prompt
  → 调模型 API（key 只在服务端）
  → 解析模型输出为「输出契约」结构
  → 回结构化 JSON 给前端渲染卡片
```

**那为什么还要后端、不让前端直连模型？** 三个理由，每个都成立：

1. **藏 API key**：放进浏览器等于公开泄露，必须在服务端调用。
2. **集中 prompt**：决断式口吻 / 营养理由 / 输出格式约束是产品的核心资产，集中在后端可统一迭代、收 badcase，不随前端发版。
3. **为演进留接口**：M2 的画像存储与跨次记忆、未来账号体系，都挂在这个网关后面，前端契约不变。

## 3 · 请求 / 响应与输出契约解析

单轮、单次模型调用，对应 [[1.0方案]] 的核心闭环。

- **请求**：前端 POST `{ profile, todayMeals, mood? }`。
- **拼装**：系统角色（决断式营养顾问）+ 画像 + 今日已吃 + 当下需求 + **输出格式约束**。
- **输出**：要求模型返回**严格 JSON**，而非自由文本——
  ```json
  { "dish": "…", "reason": "…", "buy": "…", "cook": "…" }
  ```
  `dish` 对应输出契约第一行「今晚吃 XXXX。」，`reason` 对应第二行营养理由；`buy`/`cook` 供「去哪买 / 怎么做」按需取用。**由后端拼成最终文案**，保证「决断、不给选项清单」的呈现由我们控制，不靠模型自觉。
- **可靠性**：后端校验 JSON 是否合法、字段是否齐全；不合法则**重试一次**（更强约束的提示），仍失败回友好兜底文案。DeepSeek 支持 JSON 输出，可显式开启降低坏格式率。
- **不流式**：1.0 输出极短（一道菜 + 一句话），流式叠加 Serverless 反而麻烦、零收益。普通请求/响应即可。

## 4 · 模型选型（收口 Q1）

判据：**中文质量 / 价格 / 延迟 / JSON 可控 / 国内可达**。本产品单次调用极轻（输入数百 token、输出约百 token），成本与延迟都不是瓶颈，重点是中文口语 + 决断口吻 + 可控 JSON。

价格为 2026-06 官网核实值（每百万 token）：

| 模型 | 角色 | 输入 | 输出 | 说明 |
| --- | --- | --- | --- | --- |
| **DeepSeek v4-flash** | **1.0 默认** | $0.14（命中缓存 $0.0028） | $0.28 | 1M 上下文，OpenAI 兼容，JSON 输出，国内可达；单次推荐成本可忽略 |
| DeepSeek v4-pro | 质量档 | $0.435 | $0.87 | 同 API 一行换；理由/口吻不够好时升档 |
| 通义千问（flash 档） | 可用性兜底 | 以官网为准 | 以官网为准 | 异厂商，OpenAI 兼容；DeepSeek 故障时切换，避免单点 |
| Claude Opus 4.8 / Haiku 4.5 | 已评估，**defer** | $5 / $1 | $25 / $5 | 质量强，但国内云出网不可靠可达（§1）；接入需海外出口，违背备案+延迟，不进 1.0 |

> 注：DeepSeek 旧名 `deepseek-chat` / `deepseek-reasoner` 将于 2026-07-24 弃用，直接用 `v4-flash` / `v4-pro`。

**接入方式：adapter = 配置，不是抽象层。** 三家模型都走 OpenAI 兼容协议，所以「换模型」= 改三个环境变量（`base_url` / `model` / `api_key`）。用官方 OpenAI 兼容 SDK 直连即可，**不建框架式抽象**——换模型的需求由配置满足，过度抽象是镀金。

## 5 · 技术栈与存储

- **前端**：阿里云 OSS 静态网站托管 + CDN。单页轻交互（输入框 + 结果卡片 + 「换一个」），契合「网页优先」。
- **后端**：阿里云函数计算 FC，**单个 HTTP 触发函数**，**Node.js**（与前端同语言、生态轻、冷启动友好、OpenAI 兼容 SDK 成熟）。
- **密钥**：模型 key 存 FC 环境变量 / 凭据管理，绝不进前端、不进仓库。
- **存储（1.0）**：服务端**零存储**。画像与历史在浏览器 localStorage（收口 Q3/Q4：无登录、换设备即丢，1.0 接受）。
- **存储（M2 演进）**：加记忆时引入轻量 KV（如表格存储 Tablestore / Redis），按**匿名 device id** 存画像与吃饭历史；账号体系再上一层。前端契约不变，平滑接到 §2 的网关后面。
