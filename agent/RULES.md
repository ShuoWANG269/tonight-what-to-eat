# 今晚吃什么 · 规则

> 仅写本 project 专属硬规则。vault 级规则见 [[agent/RULES]]，全局在 `~/.codex/RULES.md`，本文件不重复。

## 产品边界

- 1.0 版本只做核心功能：一句话输入早/午餐 → 决断式晚餐推荐（一个明确答案 + 精炼理由）；「拍照识别」等不纳入 1.0
- 输入要「轻」、输出要「决断」：不给选项清单，给一个明确答案
- 衍生功能（怎么做/去哪买、放纵博弈、二次传播趣味输出）属增量，不挤占核心链路的稳定性

## 内容边界

- 涉及营养 / 热量的研判是产品功能性建议，不是医疗诊断；输出措辞避免医疗承诺
- 「阴阳暗讽」「趣味盲盒」等二次传播口吻只用于明显不合理输入，且不针对用户身体特征做冒犯性表达
