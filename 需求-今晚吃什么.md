# 需求 · 今晚吃什么

> 本 project 的初始产品需求草稿。产品定位：解决用户「今晚该吃什么」的选择困难，给出决断式晚餐推荐。
> 项目入口见 [[今晚吃什么]]。

# 一.用户初始信息输入

1.身高体重

2.性别

3.年龄

4.等等。。。

#### tips：

- 在保证推荐准确的前提下，尽可能少的设置信息输入
    

# 二.产品核心功能

解决用户困惑的“今天晚上该吃什么这一问题”。

用户输入自己早上，中午吃的饭（比如吉野家牛肉饭），产品给出用户答案“晚上该吃什么”

给出答案的依据可以是：

用户数据：用户初始信息输入+用户过去对话的记忆。比如“在用户的身高体重性别下，吃什么合适？”“用户过去总吃麦当劳，这周没吃，是不是该吃一顿？”

营养均衡：通过用户说的早上，中午的饮食，判断少了哪些营养，该吃点啥？

用户此时心情或口味：用户输入“嘴里没味”“心情不好”等需求，也要综合考虑给出用户晚上吃什么的建议。

#### tips：

- 输入要"轻"，早/中餐用一句话自然语言输入即可。可支持"拍照识别"（拍照不作为1.0版本的功能考虑）
    
- 输出要"决断"：选择困难症用户要的是一个明确答案。
    

输出样例：

第一行：“今晚吃XXXX。”

第二行：“营养角度原因” 原因要精炼

➡️不想吃？ ——换一个

➡️想吃？ ——去哪买/怎么做

# 三.产品衍生功能

1.根据推荐好的晚上吃什么，进一步推荐饭菜该怎么做，或者去什么饭店买？

2.加点博弈思路，人不是机器，不可能永远按照规范进行饮食。

- 回答“如果想放纵，怎么放纵最划算这一问题”
    

_已知今天吃了什么 → 找出今天哪些营养/热量还有"余量" → 推荐一个踩在余量上的放纵选项_

例如“主食和油today够了，想解馋就来点烤串(少主食)”

- 对用户提出的放纵策略，合理研判
    

比如用户说想喝奶茶，AI要分析今天喝奶茶是轻微影响，还是恶劣影响。

给出补救措施：_饭后散步 20 分钟，这杯奶茶就当没喝过 。_或者直接赦免_"人生哪能天天清汤寡水，今天就放纵一下，开心最重要 "_

3.

# 四.二次传播

输出内容要准确，并且随机产生趣味性答案，有助于第一批体验用户自发二次传播。

1. 如果用户输入较为抽象不合理，如“我今天上午吃了10个汉堡”，输出可以是偏阴阳，暗讽的口吻，来暗示吃的多
    
2. 比如 统计百大UP最喜欢吃的饭，如果和今晚推荐观众的饭对上了，就同输出该UP的吃饭寄语。
